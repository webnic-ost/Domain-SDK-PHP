◆ getExtensionsPrice()

Webnic\WebnicSDK\Services\DomainProduct::getExtensionsPrice ( array $filtersData)

Retrieves pricing details for domain extensions.

This function allows you to fetch standard pricing details for domain extensions based on specified filters. Note that promotional prices will not be included in the response.

Parameters
array$filtersDataAn array of filters and pagination options to refine the search. Available keys:
  • filters (array): Search conditions. Each condition should include:
    • field (string): The field to filter by. Supported values:
      • productKey (required): Filter for TLDs. Use comma-separated values for multiple TLDs.
      • transtype (optional): Filter for specific transaction types. Accepted values:
        • register - Domain registration.
        • renew - Domain renewal.
        • transfer - Domain transfer.
        • restore - Domain restoration.
        • proxy - Proxy service.
        • whois - WHOIS privacy service.
        • rereg - Re-registration service.
    • value (string): The corresponding filter value.
  • pagination (array): Pagination options for the response:
    • page (int): The page number to retrieve. Defaults to 1.
    • pageSize (int): The number of records per page. Defaults to 10.
Returns
array An associative array containing the pricing details for the requested extensions.

Retrieves promotional pricing details for domain extensions.

This function allows you to fetch promotional pricing details for specific domain extensions based on the provided filters. Prices are shown in USD. Each promotional item's conditions and effective periods (in GMT timezone) should be reviewed for accuracy.

Parameters
array$filtersDataAn array of filters to refine the promotional pricing search. Available keys:

    • register - Promotional pricing for domain registration.
    • renewal - Promotional pricing for domain renewal.
    • transfer - Promotional pricing for domain transfer.
  • ext (string): Filter by TLD (Top-Level Domain). Example: "com" or "org".
Returns
array An associative array containing promotional pricing details for the requested extensions.

Retrieves domain extension rules based on the specified rule type(RG),(TF),(GRACE)(DOC).

This function fetches rules for a given domain extension (ext) and rule type (ruleType), such as registration, transfer, grace period, or document upload rules.

Parameters
string$extThe domain extension (e.g., '.com', '.org').
string$ruleTypeThe type of rule to retrieve. Accepted values are:
  • 'registration' - Rules for domain registration.
  • 'transfer' - Rules for domain transfer.
  • 'grace' - Rules for grace period after domain expiration.
  • 'DOCUPLOAD' - Rules related to document uploads.
string$category[Optional] Required for the 'DOCUPLOAD' rule type. Specifies the category of the document (e.g., 'ID', 'Address Proof'). Default is an empty string.
bool$isProxy[Optional] Indicates if the request is made via a proxy. Applicable only for the 'DOCUPLOAD' rule type. Default is false.
Returns
array An associative array containing the rules for the specified domain extension and rule type.
Exceptions

Exception If the request fails or the provided parameters are invalid.

Usage Examples:

// Fetch registration rules for '.com' domains.
$rules = $this->getExtensionsRule('.com', 'registration');
// Fetch document upload rules for '.org' domains with category and proxy.
$rules = $this->getExtensionsRule('.org', 'DOCUPLOAD', 'ID', true);

$sortBy = null )

Smart Query that checks domain availability and provides TLD suggestions in one request.

This function performs two actions in one request:

  1. Checks the availability of a given domain name.
  2. Provides a list of available TLDs (up to 10) for the domain.

It also checks the availability on the specified TLD list and WebNIC’s top-selling TLDs, with results returned within 5 seconds.

Process Workflow:

  • Availability check on the domain name.
  • Availability check on the provided TLD list.
  • Availability check on WebNIC's top-selling TLDs.
Parameters
string$domainNameThe domain name to be checked for availability (e.g., '007.com').
string | null$sortByA list of TLDs to prioritize in the availability search. (e.g., 'my', 'events', etc.). If not provided, WebNIC's top-selling TLDs will be used.
Returns
array An associative array containing the availability status of the domain and a list of suggested TLDs (up to 10).
Exceptions

Exception If the request fails or the parameters are invalid.
